# 목표 동작(target behavior) 템플릿 — 블랙박스 E2E

> `target-behavior` 규약으로 strategist가 *수용 기준에서* 작성(구현 설계문서 미참조). 사람 행동 단위 승인 후 동결, **에이전트 비공개 경로(`_target_behavior/`)**에 저장. 평가자만 실행.

## 행동 명세 (승인란)
| # | 입력(요청) | 기대(상태·결과) | 분류 | 승인 |
|---|-----------|----------------|------|------|
| 1 | 정상 생성 | 201 + 리소스 | 정상 | [ ] |
| 2 | 경계(예: 깊이 N+1) | 400/규약 | 경계 | [ ] |
| 3 | **남의 리소스 참조** | **404** | 권한 | [ ] |
| 4 | 검증 실패 | 400 + {code,message,errors[]} | 에러계약 | [ ] |
| 5 | (고위험) 동시 요청 | 중복 없음/규약 | 동시성 | [ ] |

## E2E 골격 (스택 등가물)
```
// _target_behavior/{Api}TargetBehaviorE2ETest — 내부 DTO 비의존, HTTP 표면만
// given: 인증 사용자 A, 사용자 B의 리소스
// when:  A가 B의 리소스에 접근 요청
// then:  HTTP 404
```

## 규칙
- 블랙박스만(내부 구현·DTO 비의존). 동결 후 변경 = 계약 변경 = 사람 결정.
