# 기능 입력 템플릿 (인스턴스)

> 개발자가 한 기능을 시작할 때 작성. 자유 서술 + **필수 위험칸**(신입이 잘 빠뜨리는 곳 — 미입력 시 진행 불가).

## 기능 (자유 서술)
- 무엇을: <예: 리소스 생성 API (POST /api/v1/{resource})>
- 왜/맥락: <...>
- 입력·출력 개요: <요청 바디 / 성공 응답>

## ★ 필수 위험칸 (비우면 시작 안 됨)
- **권한·소유권**: 누가 할 수 있나? 남의 리소스 접근 시? <예: 로그인 사용자만; 남의 리소스 접근 → 거부>
- **동시성**: 동시에 오면 위험한 연산 있나? <예: 태그 find-or-create 동시 생성 → 중복 금지 / 없음>
- **에러 계약**: 실패 케이스와 상태코드·에러형식 <예: 검증실패 400, 없음 404, 형식 {code,message,errors[]}>

## 경계·예외 (선택 — 비우면 strategist가 초안)
- <깊이 제한 / 빈 값 / 최대 길이 / soft delete 상호작용 ...>

## 비고
- 심층 검사(뮤테이션·성능·이중 목표 동작)는 위 위험칸·risk-profile 기준으로 *자동* 적용된다 — 모드 선택 없음.
