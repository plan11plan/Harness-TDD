# TDD Autopilot Harness — 오케스트레이터

비-LLM 운전자. 역할 에이전트를 호출하고, **게이트(`locked-evaluator`)를 직접 실행**해 결과 코드로 다음 단계를 정한다. 컨텍스트는 (API×계층)마다 리셋하고 신뢰 원장·작업 브랜치 커밋에서 복원한다.

## 입력
- `slots/`(conventions·stack.manifest·risk-profile) + 기능 입력(`templates/feature-input.template.md`).
- **git 설정**도 `stack.manifest`의 `git:` 블록에서 온다 — `base_branch` · `branch_prefix` · `branch_cmd`/`commit_cmd`/`pr_cmd` · `auto_merge: false`(+선택 `draft_pr`). 엔진은 특정 VCS/호스트 명령을 모르므로 *전부 슬롯에서* 읽는다.
- 매니페스트가 비면 진행 불가(추론 금지) → 사람에게.

## Phase 0 — Pre-flight 점검 (잠그기 전, strategist + 사람)
strategist가 에스컬레이션 트리거 체크리스트를 *미리* 돌린다: 명세 모호/충돌 · 빠진 규칙 · 고위험 연산 · 엣지 누락. 발견을 사람과 해소 → 계약을 최대한 완전하게. (목적: 주행 중 멈춤을 사전에 제거.)

## Phase 1 — 사전 전략·잠금 (strategist, 1회, ★동기 관문)
1. 리스크 우선순위·계층 비율·커버리지 바닥·CI 설계 산출.
2. `target-behavior`로 **목표 동작 초안**(수용기준에서, 구현 설계문서 미참조).
3. **★사람 행동 단위 승인** → 목표 동작·임계값 **동결**(에이전트 비공개 경로). = 인터페이스 C의 동기 관문 ①.

## Phase 2 — 루프 (기능마다, 레벨 단위→통합→E2E 순)

### 루프 진입 — 작업 브랜치 생성 (기능/계약당 1회)
계약 동결 직후, 오케스트레이터가 `git.base_branch`에서 작업 브랜치 `{git.branch_prefix}/{기능}`을 분기한다(`git.branch_cmd`). **이후 모든 커밋은 이 브랜치에만 쌓이고, base/메인엔 직접 커밋하지 않는다.** *(옵션 `git.draft_pr: true`면 이 시점에 Draft PR을 열어 커밋 스트림으로 진행을 실시간 노출.)*

**테스트 레벨 단위로 돈다** (별도 레벨 에이전트 없음 — 같은 역할 에이전트가 레벨마다 그 레벨 테스트를 만든다):
- 단위 → 의존성 격리 / 통합 → 실제 인프라 / E2E → 엔드투엔드(실제 프로토콜·인증).
- **레벨 ↔ 코드 위치·테스트 패턴은 매니페스트 `levels:`**, 격리 방식·계층명은 `project-conventions` 슬롯이 정한다(엔진은 특정 레이어명·프레임워크를 모른다). **3레벨 모두 필수**(DoD) — 오케스트레이터가 3레벨을 돌고 auditor가 존재를 검사해 강제.

designer가 (깊이는 리스크 비례) GWT 케이스 산출 후, 레벨마다 RGR 증분을 돈다. **커밋은 각 게이트를 통과한 뒤에만** 찍는다(정책 b — 아래):

| 단계 | 누가 | 게이트 → 커밋 |
|------|------|--------------|
| 테스트 작성 | red-author | (미커밋) 실패 테스트 + 컴파일 골격 |
| RED 검증 | 오케스트레이터 | `{test_cmd}` → 게이트1(RED) |
| **R 커밋** | 오케스트레이터 | 게이트1 통과·**동결 후** 테스트+골격 커밋 (구현보다 선행 = 게이트2 순서) |
| 구현 | green-implementer | (자동수정 ≤3) |
| GREEN 검증 | 오케스트레이터 | 게이트3(GREEN) |
| **G 커밋** | 오케스트레이터 | 게이트3 통과 후 구현 커밋 |
| 리팩터 | refactorer | |
| 유지 검증 | 오케스트레이터 | 유지 게이트 (+게이트4 객관 룰, *arch_cmd 있을 때만*) |
| **R 커밋** | 오케스트레이터 | 유지 통과 후 리팩터 커밋 + **원장 1행** |
| Decide | 오케스트레이터 | 전진 / revert / 자동복구 / 에스컬레이션 |

기능의 E2E 레벨까지 끝나면 **게이트5(커버리지)·6(동시성)·7(목표 동작)·8(심층, 고위험 자동)**.

> **커밋 정책 (b — 게이트 후 커밋, 작업 브랜치 자동):** 각 RGR 증분에서 **R / G / R(refactor)을 별도 커밋**으로 작업 브랜치에 쌓되, **각 커밋은 해당 게이트를 통과한 뒤에만** 찍는다. 즉 RED 게이트로 *깨끗한 RED*가 확인되고 테스트가 **동결된 뒤** 테스트 커밋을 찍고(→ 이후 `src/test/**` 변경 금지, 구현 커밋보다 선행 = 게이트2 순서 증거), GREEN 통과 후 구현 커밋, 유지 게이트 통과 후 리팩터 커밋(+원장 1행)을 찍는다. **게이트에 반려된 시도(불결한 RED·실패한 GREEN·빨강 리팩터)는 커밋하지 않고 워킹트리에서 고친다** → 실패 시도가 히스토리에 남지 않아 PR이 깨끗하고, `test SHA < impl SHA` 순서는 항상 유지된다. 원장 파일(`trust-ledger.md`)도 같은 브랜치에 동반 커밋. 자동복구로 추가된 RED도 동일 규칙. **메인 직접 커밋·자동 머지는 하지 않는다.**

## 자동 복구 vs 에스컬레이션 (라우팅 핵심)

> **계약 안에서의 실패 = 자동 복구. 계약 자체를 바꿔야 하면 = 사람.**

- **자동 복구(안 멈춤, 보고만):** 목표 동작이 *명시된* 행동 누락을 잡음 / 커버리지가 *명시 경로* 미달 / RED·GREEN 게이트 기계 반려.
  → 실패를 **새 RED 케이스로 피드백** → 정식 RGR 증분으로 보강 → 원장에 "자동 보강" 기록. **계약(목표 동작·동결 테스트·전략)은 절대 수정 않고 *충족*만.**
- **에스컬레이션(인박스, 사람):** 명세 모호/충돌 · 동결 테스트 모순 · 새 아키텍처 결정 · 목표 동작/마이그레이션 승인 · 환경/자격증명 부재 · **자동복구 N=3회 소진**.

## 인터페이스 (C — 비동기 주행 + 동기 계약 관문)
- 기본: 비동기. 기능(들)을 큐에 넣고 백그라운드 주행(각자 작업 브랜치). 에스컬레이션은 **인박스**에 쌓고 알림 → 사람이 틈날 때 응답 → 재개.
- **동기 관문 2개:** ①목표 동작 승인(Phase 1) ②파괴적 마이그레이션 범위 승인. 이 둘만 블록.
- **완료 = PR.** 감사 PASS 후 오케스트레이터가 **PR을 생성**한다(`git.pr_cmd`, base ← 작업 브랜치). 본문은 `templates/pr-body.template.md` — **원장 요약 · 감사 판정 · 게이트 통과표 · R/G/R 커밋·`git diff` 링크**. 사람은 PR 하나로 진행·결과물·증거를 확인한다(완료 리뷰 관문).
- **머지는 사람만.** 승인 → 사람이 머지 / 변경요청 → **같은 브랜치에서 재실행**(추가 커밋) → PR 자동 갱신 → 재리뷰. **자동 머지·메인 직접 커밋 금지.**

## Phase 3 — 감사 → PR + 주기 응집
전 계층 초록 후 `tdd-auditor`(품질·정합성·NITS). FAIL이면 지시 역할 1회 재호출 후 **같은 브랜치에서** 재작업·재감사. **PASS면 오케스트레이터가 PR을 생성/(Draft였으면) Ready 전환**(인터페이스 C — 사람 리뷰·머지 관문). **마일스톤마다 `tdd-coherence-auditor`**(교차 중복·도메인 응집·죽은 코드 → 리팩터 RED 제안 = 글로벌 응집 교정).

## 신뢰 원장 (파일 + git, 둘 다)
- **파일:** `templates/trust-ledger.template.md` 스키마로 증분당 1행(RGR 전 단계 + 자동복구·에스컬레이션·개입·무인완주). 작업 브랜치에 동반 커밋되고 PR 본문에 요약된다.
- **git:** 작업 브랜치의 **R/G/R 별도 커밋(SHA)**이 증거(테스트 커밋 선행 = 순서 증거). **컨텍스트 복원의 원천** — (API×계층) 리셋 후 브랜치 커밋·원장에서 복원.
- **읽는 주체 / 시점:** 오케스트레이터(증분 시작 복원) · `tdd-auditor`(RED 정직성 교차확인) · 사람(PR 본문 요약 리뷰).

## Error Handling
| 상황 | 전략 |
|------|------|
| 게이트 반려 | 해당 역할 재호출(사유별), ≤N |
| 자동복구 N 소진 | 에스컬레이션(조용한 통과 금지) |
| 매니페스트/슬롯 누락 | 진행 불가 → 사람 |
| 외부 자격증명 부재 | 에스컬레이션(목/스킵 표시) |
| PR 생성/푸시 실패(인증·권한) | 에스컬레이션(브랜치·커밋은 보존, 원장에 기록) |

> 미해결은 **PR 본문(과 원장)**에 명시. **작업 브랜치 자동 커밋은 허용**(R/G/R 별도, 증거). **메인 직접 커밋·자동 머지는 금지** — base 반영은 PR로 사람이 승인·머지한다.

## Modes
- **표준(항상):** 전 파이프라인. designer 깊이·심층 검사(뮤테이션·N+1/성능·이중 목표 동작)를 **risk-profile 리스크에 비례해 자동 적용** — 고위험은 깊게, 사소한 경로는 얕게. *별도 deep 토글 없음.*
- **재실행:** "이 API만 다시" → 해당 증분 역할만 재호출(같은 작업 브랜치에 추가 커밋 → PR 갱신).
