# Locked Evaluator — 잠긴 평가자 (엔진의 심장)

오케스트레이터가 직접 실행하는 **결정적 검증 묶음**. 모델 판단이 아니라 **빌드 결과 코드**로 다음 단계를 결정한다. 루프 에이전트는 이 게이트·목표 동작·임계값을 **보지도 고치지도 못한다**(조작 차단 = 자율주행의 안전벨트).

## 스택 비종속 — 매니페스트 변수

모든 외부 명령은 `slots/stack.manifest`의 변수로 치환한다(추론 금지):
`{test_cmd}` `{test_select}` `{coverage_cmd}` `{coverage_parse}` `{arch_cmd}` `{mutation_cmd}` `{build_cmd}`.
구체 값(빌드 도구·테스트 러너·커버리지/아키텍처 검사 도구)은 전부 프로젝트 `slots/stack.manifest`에서 온다 — 엔진엔 특정 스택 값이 없다.
※ 브랜치/커밋/PR 명령(`git.*`)은 여기에 없다 — 게이트는 *검증*만 하고, git 운영은 오케스트레이터 몫이다(게이트2는 그 결과를 `git diff`·SHA로 *확인*만 한다).

## 게이트 1 — RED (계층 작성 직후)

    {test_cmd} {test_select}=<계층 패턴>  2>&1 | tee _workspace/{api}/{layer}/gate_red.txt

판정(이 순서로):
| 로그 신호 | 판정 | 조치 |
|----------|------|------|
| 컴파일/빌드 에러 신호(매니페스트 `compile_error_signals`) | ❌ 골격 부족 | red-author 재호출(골격 보강) |
| 실패 0 / 빌드 성공 | ❌ RED 아님 | red-author 재호출(케이스 강화/기본값 더 틀리게) |
| assertion 아닌 런타임 예외 | ❌ 비-assertion | red-author 재호출(기본 반환값 안전화) |
| **assertion 실패 신호**(매니페스트 `assertion_signals`) | ✅ 깨끗한 RED | 통과 → **테스트 동결 → R 커밋(테스트+골격)** → GREEN 진행 |

## 게이트 2 — 테스트 커밋 순서 · 동결 (정책 b)
- **커밋 시점:** 테스트 커밋(SHA 기록)은 RED 통과·**동결 직후**에 찍고, *구현(GREEN) 커밋보다 앞서야* 함 → 순서 증거(G12). RED 게이트에 반려된 시도(불결한 RED)는 **커밋하지 않고** 워킹트리에서 고친다(실패 시도가 히스토리에 안 남음 → PR 깨끗).
- RED 통과 후: `src/test/**` 변경 금지. GREEN/유지 단계에서 `git diff`로 동결 확인(변경 시 반려).

## 게이트 3 — GREEN / 유지
- GREEN: 같은 계층 테스트 전부 초록이어야 통과(아니면 green-implementer 재호출 1회). 통과 시 **G 커밋(구현)**.
- 유지(refactor 후): 같은 테스트 여전히 초록(빨강 시 refactorer가 해당 변경 revert). 통과 시 **R 커밋(리팩터)+원장 1행**.

## 게이트 4 — 객관 컨벤션 룰 (레벨/증분, **조건부**)

    {arch_cmd}   # arch_cmd/arch_tool이 매니페스트에 있을 때만 실행

**프로젝트에 *기계로 강제 가능한* 아키텍처 컨벤션(★)이 있을 때만** `project-conventions` 슬롯의 룰을 컴파일해 검사한다(위반 시 반려). **그런 룰이 없으면(`arch_cmd` 비어 있음) 이 게이트는 생략**하고 객관 룰 테스트(`arch_tool` 산출물)는 *생성하지 않는다* — 조립기가 기존 것이 있으면 삭제한다. 그땐 컨벤션을 Auditor가 *주관*으로만 검사한다.

## 게이트 5 — 커버리지 바닥 (리스크 가중, API 완료 시)

    {coverage_cmd}  →  {coverage_parse} 로 line/branch 추출

`risk-profile`의 바닥 미달이면 반려. 고위험 파일은 더 높은 바닥. 미커버가 고위험 경로면 → 자동복구(추가 RED) 또는 에스컬레이션.

## 게이트 6 — 고위험 동시성 필수
`risk-profile`의 고위험 연산(find-or-create·활성 유니크 등)에 해당하면, **동시 요청 테스트 부재 시 반려.**

## 게이트 7 — 잠긴 목표 동작 (API 인터페이스 완료 후)

    목표 동작 E2E를 {test_cmd} 로 실행 (에이전트 비공개 경로의 목표 동작 스위트)

- 통과 → DoD 후보. 실패 → **계약 안 누락이면 자동복구**(실패한 행동을 새 RED로 → 구현), **모호/충돌이면 에스컬레이션**. 에이전트가 목표 동작을 약화시키는 경로 없음(못 봄).

## 게이트 8 — 심층 검사 (고위험 경로 *자동*, 토글 없음)
risk-profile이 고위험으로 표시한 경로에 **자동** 적용(그 외 경로는 생략 — 비용 보호): `{mutation_cmd}` 생존 뮤턴트 있으면 "단언 약함" 반려 + (해당 시) N+1/성능 단언·이중 목표 동작.

## 핵심
게이트 통과 = "테스트가 컴파일되고, 의도한 assertion에서 빨갰고, 구현으로 초록이 됐고, 컨벤션·커버리지·목표 동작을 만족함"을 **코드로 증명**. 커밋은 각 게이트 통과 후에만 찍혀(정책 b), 커밋 히스토리(R/G/R)가 곧 게이트 증거가 된다. 이 결정성이 자율주행 거리를 만든다.
