# 리스크 프로필 (프로젝트별 슬롯) — 면훈소 예시

> strategist가 커버리지 바닥·동시성 필수·목표 동작 두께를 정하는 근거.

## 고위험 연산 (동시성 테스트 필수 + 커버리지 바닥↑)
| 연산 | 리스크 | 필수 검증 |
|------|--------|----------|
| 태그 find-or-create | 동시 생성 중복 | 동시 요청 테스트 |
| 활성 유니크(카테고리/질문) | 경합 시 중복 활성 | 동시 요청 + soft delete 상호작용 |
| 재귀 꼬리질문 트리(깊이≤N) | 깊이 우회·순환 | 경계(깊이 N, N+1)·순환 방지 |
| 권한·소유권(전 엔드포인트) | 남의 리소스 접근 | 남의 리소스 → 404 (목표 동작 필수) |

## 커버리지 바닥 (리스크 가중)
| 대상 | line | branch |
|------|------|--------|
| 기본(전체) | 80% | 70% |
| 고위험 파일(위 연산 포함) | 90% | 85% |
| DTO·설정·상수 | 제외 | 제외 |

## 머지 조건
- 바닥 미달 → 자동복구(명시 경로 추가 RED) 또는 에스컬레이션. 전체 빌드 초록 + 감사 PASS(또는 NITS) + 미해결 에스컬레이션 0.

## 심층 검사 자동 적용 대상 (뮤테이션·성능·이중 목표 동작)
권한·동시성·금전성 핵심 경로에 **자동** 적용(토글 없음). 그 외 경로는 생략.
