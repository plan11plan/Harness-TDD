## 1. 기능 계약 (수용 기준)
- **요청**: `{ parentId?, content(1~2000, 필수), modelAnswer?(0~5000), categoryId(필수), tagNames?[], visibility?(기본 PRIVATE) }`
- **성공**: `201 Created` + `Location: /api/v1/questions/{id}` + 본문 `QuestionDetail`
- **에러**: `400 VALIDATION_ERROR`(내용 빈값/길이초과) · `404 CATEGORY_NOT_FOUND`(없는/남의 카테고리) · `404 PARENT_NOT_FOUND`(없는/남의 부모) · `401`(미인증)
- **태그**: `tagNames` 자유텍스트 → 내 태그에서 find-or-create
- **소유권**: 모든 참조(category·parent)는 owner 스코프, 위반 시 404 통일

## 2. 리스크 우선순위 (risk-profile.md 매핑)
| 연산 | 리스크 | 필수 검증 | 레벨 |
|------|--------|----------|------|
| 권한·소유권(category/parent) | 남의 리소스 접근 | 남의 카테고리/부모 → 404 | 통합·E2E·목표동작 |
| 태그 find-or-create | 동시 생성 중복 | **동시 요청 테스트**(게이트6) | 통합 |
| content 불변식 | 빈값·과길이 우회 | 경계(빈값, 2000/2001) | 단위·E2E·목표동작 |

## 3. 계층 비율 / 커버리지 바닥 (risk-profile)
- 단위(domain): 불변식·부모검증 — fake repo. 바닥 line 90 / branch 85 (고위험: 소유권·불변식 포함)
- 통합(application): 카테고리 소유권 + 태그 조합 + **find-or-create 동시성** — Testcontainers MySQL
- E2E(interface): 201/400/401/404 — RANDOM_PORT + 인증
- 전체 question 패키지 바닥: line ≥ 80 / branch ≥ 70 (DTO·설정 제외), 고위험 파일 90/85

## 4. GWT 케이스 (designer 산출)
- **단위**: 루트 생성 저장 / 부모 있으면 꼬리질문 / 없는 부모 → PARENT_NOT_FOUND / 빈 내용 → VALIDATION_ERROR / 2001자 → VALIDATION_ERROR / visibility null → PRIVATE
- **통합**: 카테고리 소유 검증 후 저장+태그연결 / 남의 카테고리 → CATEGORY_NOT_FOUND / tagNames find-or-create(중복 이름 1개) / **동시 생성 시 같은 태그 1개만 생성(동시성)**
- **E2E**: 201+Location+상세 / 400(빈 내용) / 401(미인증) / 404(남의 카테고리)

## 6. 가정·결정 (assumptions)
- **A1. base 브랜치 = `exp/results`** — main엔 의존 도메인(user/category/tag)이 없어 컴파일 불가. 블로그 PR "→ main"은 실제 base로 정정.
- **A2. 격리 워크트리** — 사용자의 더티 트리(하네스 재구성 중) 보호 위해 `harness/question-create` worktree에서 실행.
- **A3. 목표 동작 위치** — 멀티에이전트 본주행에선 `_target_behavior/`(루프 비공개). 이 단일-오케스트레이터 evidence run에선 gradle 실행을 위해 test 소스셋(`question/targetbehavior/`)에 두되 **스펙으로만 작성·동결**해 의미(독립 합격·비조정)를 보존.
- **A4. 게이트4(객관 룰/ArchUnit)·게이트8(뮤테이션) 생략** — 프로젝트에 ArchUnitTest·PIT 미설정(`arch_cmd`/`mutation_cmd` 비움). 컨벤션은 Auditor 주관 검사.
- **A5. 범위 = 생성(CREATE) 유스케이스 1개** — 목록/상세/수정/삭제는 본 evidence run 범위 외(불필요 코드 배제 → 커버리지 정확).
- **A6. 사람 승인 관문** — 본 evidence run은 사용자가 기능 선정으로 사전 승인한 것으로 간주(동기 관문 ①).
